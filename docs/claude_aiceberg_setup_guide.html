Claude + Aiceberg Setup Guide

This guide explains how to add Aiceberg observability and control to your Claude agents. It works for both the Python SDK and the Claude Code CLI.

What Does This Do?

Once installed, every interaction with Claude is tracked on the Aiceberg dashboard:

What gets trackedHow
User promptsOpens a user_agt event, closes it with the assistant's response
Tool calls (Read, Bash, WebSearch, …)Opens an agt_tool event on PreToolUse, closes it on PostToolUse
Subagent delegation (Agent/Task tools)Opens an agt_agt event, same open/close pattern
Blocked actionsIf Aiceberg policy blocks a prompt or tool, the agent stops and the event is closed cleanly
ErrorsAPI failures (StopFailure) close all open events so nothing is left orphaned

Prerequisites

  • Python 3.11+
  • An Aiceberg account with an API key and a Use Case ID
  • For CLI path: Claude Code installed (claude command available)

Setup (Both Paths)

1 Clone & install the package

git clone https://github.com/sravanjosh07/claude_agents_sdk.git
cd claude_agents_sdk
python -m venv .venv
source .venv/bin/activate
pip install -e .

2 Create a .env file in your project root

# Required
AICEBERG_API_KEY=your-api-key-here
USE_CASE_ID=your-use-case-id-here

# Optional
AICEBERG_DRY_RUN=false
AICEBERG_USER_ID=your-name
You can find the API key and Use Case ID in your Aiceberg account settings.

Path A — Claude Code CLI (Terminal)

This is for people who use claude in the terminal. You type prompts, Claude uses tools, and Aiceberg watches everything.

3 Generate the hook settings

cd /path/to/your-project
claude-aiceberg-init --workspace . --allow-web-search --debug

This writes .claude/settings.local.json with all the hook entries. You only need to do this once per project.

4 Use Claude as normal

claude

That's it. Every prompt and tool call now shows up on the Aiceberg dashboard automatically.

How it works under the hood

  1. Claude Code fires hook events (UserPromptSubmit, PreToolUse, Stop, etc.)
  2. Each event spawns claude-aiceberg-hook as a subprocess
  3. The hook sends the event to Aiceberg and returns a decision (allow / block)
  4. State is persisted in SQLite at .claude/aiceberg_state.sqlite3

Path B — Python SDK (Code)

This is for people writing Python agents using the Claude Agent SDK.

3 Add three lines to your code

from claude_agent_sdk import query
from claude_aiceberg import ClaudeAicebergHooks

hooks = ClaudeAicebergHooks()
options = hooks.agent_options(prompt="What is the weather in Houston?")

async for msg in query(options=options):
    # your existing message handling
    ...

4 Close the turn when done

# After the loop finishes:
await hooks.complete_user_turn(session_id, final_text)

# If something goes wrong:
await hooks.fail_session(session_id, str(exc))

With subagents

from claude_agent_sdk import AgentDefinition
from claude_aiceberg import ClaudeAicebergHooks

agents = {
    "file-counter": AgentDefinition(
        name="file-counter",
        description="Counts files matching a pattern.",
        instructions="Use Glob + Bash to count files.",
        allowed_tools=["Glob", "Bash"],
    ),
}

hooks = ClaudeAicebergHooks(agents=agents)
options = hooks.agent_options(prompt="Count all Python files in this repo.")

A ready-to-run example is at examples/run_hooks.py.

What Gets Sent to Aiceberg

Hook EventEvent TypeBlockable?Notes
UserPromptSubmituser_agt✅ YesOpens when user sends a prompt
PreToolUseagt_tool / agt_agt✅ YesOpens when Claude wants to use a tool
PostToolUse—NoCloses the tool event with the result
PostToolUseFailure—NoCloses the tool event with an error
Stop—NoCloses the user event with assistant response
StopFailure—NoCloses everything on API errors
SubagentStart—NoRegisters subagent identity locally
SubagentStop—NoRecords stop time and transcript path
Blocking: If Aiceberg policy says "block", the agent stops immediately. The prompt or tool call never reaches Claude. The event is closed on the dashboard with a block message.

Troubleshooting

ProblemFix
Events not showing upCheck AICEBERG_DRY_RUN is false and API key is set in .env
claude-aiceberg-hook: command not foundRun pip install -e . again — the console scripts need an editable install
Events stay "open" on dashboardMake sure you call complete_user_turn() (SDK) or that Stop hook is firing (CLI)
Need debug logsSet AICEBERG_HOOK_DEBUG=1 in .env — logs go to .claude/aiceberg_hook_debug.jsonl
Settings file looks wrongDelete .claude/settings.local.json and re-run claude-aiceberg-init --workspace . --debug
